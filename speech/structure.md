# 结构提纲规范

> 用途：规定演讲稿的章节顺序和内容要点，确保逻辑清晰、重点突出。

## 时长约束
- 总时长：3分钟（约900字，语速300字/分钟）

## 结构安排

### 1. 开场介绍（约30秒，150字）
- 问候与自我介绍
- 项目名称和核心定位
- 一句话概括项目价值

### 2. 项目背景与痛点（约40秒，200字）
- 当前AI助手的普遍不足
- 我们要解决的核心问题
- 项目的创新切入点

### 3. 核心技术方案（约60秒，300字）
- 混合存储架构（SQLite + ChromaDB）
- 自动记忆系统工作原理
- 学习助手的SM-2算法应用

### 4. 测试成果展示（约40秒，200字）
- 关键性能指标
- 记忆准确率数据
- 核心结论总结

### 5. 总结与展望（约10秒，50字）
- 项目价值重申
- 未来发展方向

## 时间分配表

| 部分 | 时长 | 字数 |
|------|------|------|
| 开场介绍 | 30秒 | 150字 |
| 背景痛点 | 40秒 | 200字 |
| 技术方案 | 60秒 | 300字 |
| 成果展示 | 40秒 | 200字 |
| 总结展望 | 10秒 | 50字 |
| **总计** | **3分钟** | **900字** |
