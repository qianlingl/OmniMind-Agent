# OmniMind-Agent 技术规格文档

## 1. 项目背景

OmniMind-Agent 是一个多模态 AI Agent 系统，具备长上下文记忆、文件操作、联网搜索、文档阅读、长期记忆和多智能体协作能力。本系统旨在为用户提供智能化的任务处理和知识管理服务。

## 2. 核心目标

| 目标编号 | 目标描述 |
| :--- | :--- |
| GOAL-01 | 支持 64k tokens 上下文窗口，实现高效对话压缩 |
| GOAL-02 | 提供完整的文件系统操作能力（读/写/删/重命名/批量操作） |
| GOAL-03 | 集成 Bing 和 SearchAPI 搜索引擎，支持智能搜索触发 |
| GOAL-04 | 支持 PDF、Office、TXT/Markdown 文档解析及网页内容提取 |
| GOAL-05 | 实现混合存储的长期记忆系统，支持跨会话调用 |
| GOAL-06 | 构建多 Agent 协作架构（编码、测试、文档） |
| GOAL-07 | 实现低延迟响应（<1s）和高可用性（99.9%+） |
| GOAL-08 | 实现超个性化学习助手，支持目标分解、每日推送、对话测验、间隔重复、闪卡与思维导图生成 |

### 3.2 模块职责

| 模块 | 职责描述 |
| :--- | :--- |
| **对话管理** | 管理对话上下文、实现对话压缩策略、维护会话状态 |
| **文件操作** | 提供文件读写、删除、重命名、批量操作能力 |
| **联网搜索** | 集成 Bing/SearchAPI，根据触发条件执行搜索 |
| **文档解析** | 解析 PDF、Office、TXT/Markdown 及网页内容 |
| **长期记忆** | 混合存储（向量+关系），支持跨会话知识调用 |
| **任务编排** | 主管 Agent 调度子 Agent（编码、测试、文档）协作完成任务 |
| **目标分解器** | 将用户学习目标拆解为周里程碑和日任务，结合用户基础与空闲时间 |
| **测验引擎** | 生成对话式测验题，评估用户回答质量，给出评分与反馈 |
| **间隔重复引擎** | 基于 SM-2 算法和 Ebbinghaus 遗忘曲线，计算最佳复习时间点 |
| **闪卡/导图生成器** | 从用户笔记中提取知识点，生成 Q&A 闪卡和 Markdown/Mermaid 思维导图 |

## 5. 关键设计

### 5.1 对话压缩策略

- **摘要压缩**：当上下文接近 64k 限制时，自动生成对话摘要
- **关键信息保留**：保留实体信息、关键决策点和任务状态
- **分层存储**：近期对话全量存储，历史对话摘要存储

### 5.2 长期记忆架构

```
长期记忆 = 事实库 + 知识图谱 + 经验库
    ├── 事实库：结构化数据存储（关系数据库）
    ├── 知识图谱：实体关系存储（向量数据库）
    └── 经验库：过往解决方案存储（文档检索）
```

### 5.3 多 Agent 协作模式

| Agent 角色 | 职责 |
| :--- | :--- |
| **主管 Agent** | 任务分解、调度子 Agent、结果汇总 |
| **编码 Agent** | 代码编写、调试、优化 |
| **测试 Agent** | 测试用例编写、测试执行、报告生成 |
| **文档 Agent** | 文档编写、技术规范生成、文档更新 |

### 5.4 学习助手核心设计

#### 5.4.1 目标分解策略

```
用户目标输入 → 基础水平评估 → 领域知识图谱映射 → 时间约束计算 → 输出任务树

示例：
  输入: "三个月学会基础 Python，每天 1 小时"
  输出:
    ├── 第 1-2 周: Python 基础语法
    │   ├── Day 1: 变量与数据类型 (30min 阅读 + 30min 练习)
    │   ├── Day 2: 条件判断与循环 (30min 阅读 + 30min 练习)
    │   └── ...
    ├── 第 3-4 周: 函数与模块
    │   └── ...
    └── 第 5-12 周: ...
```

#### 5.4.2 间隔重复算法

基于 SM-2 (SuperMemo-2) 算法的改进版：

| 参数 | 说明 | 默认值 |
| :--- | :--- | :--- |
| `quality` | 用户回答质量评分 (0-5) | - |
| `interval` | 当前复习间隔（天） | 1 |
| `easiness` | 记忆容易度因子 (≥1.3) | 2.5 |
| `repetitions` | 连续正确次数 | 0 |

**核心公式**：
- `easiness' = easiness + 0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02)`
- 当 quality < 3: 重置 interval = 1, repetitions = 0
- 当 quality ≥ 3: interval' = interval * easiness'

**遗忘曲线推送时机**：默认复习序列为 1天 → 3天 → 7天 → 16天 → 35天

#### 5.4.3 对话式测验流程

```
1. AI 根据当前学习主题生成问题
2. 用户自由文本回答
3. LLM 评估回答：
   - 正确性评分 (0-5)
   - 关键遗漏点列表
   - 针对性反馈与补充说明
4. 更新间隔重复调度
5. 记录薄弱知识点到用户记忆
```

#### 5.4.4 闪卡与思维导图生成

- **闪卡生成**：笔记/摘录 → LLM 提取关键概念 → 生成 Q&A 对 → 存入闪卡库
- **思维导图生成**：解析文档结构 → 提取层级关系 → 生成 Mermaid mindmap 语法 → 前端渲染

### 5.5 前端设计

| 页面 | 功能 |
| :--- | :--- |
| **学习仪表盘** | 今日任务、连续学习天数、进度概览、待复习数量 |
| **目标设定向导** | 多步骤表单：主题 → 时间 → 基础评估 → 确认计划 |
| **每日任务卡片** | 任务列表，每项标注预估时长、完成状态 |
| **对话式测验界面** | 聊天式交互，AI 提问 → 用户回答 → AI 反馈评分 |
| **闪卡复习界面** | 正面问题 → 点击翻转 → 用户自评掌握度 → 调度下次复习 |
| **思维导图查看器** | Mermaid 渲染，可折叠/展开节点 |
| **笔记上传区** | 拖拽上传，支持文本粘贴和文件上传 |
| **学习报告页** | 进度曲线、掌握度热力图、薄弱点列表 |
