## 通用质量检查项（适用于所有文档）
- [ ] 主题覆盖：是否完整覆盖 topic.md 中规定的核心主题，且未出现禁止主题？
- [ ] 风格一致：全文语气、用词是否符合 voice.md 的规定？
- [ ] 结构完整：章节顺序是否符合 structure.md？有无重要章节遗漏？
- [ ] 无低级错误：是否存在错别字、错误标点、乱码？
- [ ] 逻辑自洽：前后论述是否一致，有无矛盾之处？

## 填写示例：软件工程项目报告 专项检查项
- [ ] 术语一致性：全文对同一技术概念的称呼是否前后统一？
- [ ] 图表规范性：图表是否有编号和标题？与正文是否交叉引用？
- [ ] 数据可信度：测试数据、性能指标是否标明前提条件？
- [ ] 需求覆盖度：设计、测试章节是否响应了所有已列出的需求？

## 填写示例：读书心得 专项检查项
- [ ] 原创性：个人感悟部分是否与他人评论或网文高度雷同？
- [ ] 平衡性：概述与感悟的比例是否适当（感悟部分不少于60%）？
- [ ] 引用规范：引用原文是否标注页码或章节？

## 填写示例：入党申请书 专项检查项
- [ ] 政治术语准确性：党的理论表述是否与最新文件一致？
- [ ] 个人化程度：是否结合了自身具体经历，而非通用模板？
- [ ] 态度端正：是否存在错别字、涂改痕迹（手写版）或排版乱码（电子版）？
