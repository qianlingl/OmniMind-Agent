# AI Agent 测试计划 (test.md)

## 1. 测试策略
采用分层测试模型：
- **单元测试**：各模块核心逻辑（文件操作安全校验、记忆压缩算法）
- **集成测试**：API 路径、模块间交互（搜索→引用标注→对话生成）
- **端到端测试**：真实场景模拟用户与Agent多轮交互
- **性能测试**：长上下文延迟、搜索超时、并发文件操作

## 2. 测试用例（按功能分类）

### 2.1 多轮对话与记忆
| 用例ID | 场景 | 期望结果 |
|--------|------|-----------|
| T-01.1 | 发送10000字的连续对话，验证第50轮仍能准确引用第2轮的细节 | 答案正确，延迟不超2s |
| T-01.2 | 上下文超过32k字符，检查是否生成摘要，并丢失细节可控 | 生成的摘要包含早期关键信息，允许遗忘细枝末节 |
| T-01.3 | 明确问“我刚才说的第三点是什么”，应准确复现 | 复现正确，若有遗忘则回复“抱歉，这部分已压缩” |

### 2.2 文件系统操作
| 用例ID | 场景 | 期望结果 |
|--------|------|-----------|
| T-02.1 | 批量重命名：100个文件，模式`old_(*).txt` -> `new_$1.txt` | 全部成功，无文件名冲突 |
| T-02.2 | 整理下载文件夹：按扩展名归类 | 创建jpg、pdf等子目录，文件移动正确 |
| T-02.3 | 尝试访问`/etc/passwd` | 返回权限错误，不执行 |
| T-02.4 | 并发重命名同一文件 | 第二个请求返回资源锁定错误，无数据损坏 |

### 2.3 联网搜索
| 用例ID | 场景 | 期望结果 |
|--------|------|-----------|
| T-03.1 | 提问“今天的天气”，触发搜索 | 返回包含时效性信息的回答，并引用天气网站 |
| T-03.2 | 主要搜索引擎超时，自动切换备用引擎 | 结果仍能返回，日志记录故障转移 |
| T-03.3 | 搜索结果为零，模型应回退到通用知识 | 回答明确说明“未搜索到相关最新信息，以下是基于已知知识...” |

### 2.4 文档/链接阅读
| 用例ID | 场景 | 期望结果 |
|--------|------|-----------|
| T-04.1 | 上传10MB PDF学术论文，询问摘要 | 准确提取摘要，耗时<5s |
| T-04.2 | 输入URL包含分页文章，要求提取全文 | 合并多页内容，生成完整摘要 |
| T-04.3 | 对Excel表格提问：“第二列的平均值是多少” | 正确解析表格数据并计算平均值 |
| T-04.4 | 超大文件（>50MB）上传 | 返回 413 错误，提示文件过大 |

### 2.5 长时记忆
| 用例ID | 场景 | 期望结果 |
|--------|------|-----------|
| T-05.1 | 会话A中存储“我喜欢用bullet points”，新建会话B，问一般问题 | 回复自动使用要点列表，并提示“根据你的偏好……” |
| T-05.2 | 查询记忆：“我上次说的编程语言是什么” | 返回存储的偏好，如“Python” |
| T-05.3 | 记忆冲突更新，重复存储同一主题不同值 | 记忆更新为新值，旧版本保留历史但标记为过时 |

### 2.6 多Agent协作
| 用例ID | 场景 | 期望结果 |
|--------|------|-----------|
| T-06.1 | 给主管Agent：“写一个Python函数反转字符串，并测试它” | 主管分解为Coding和Testing子任务，最终返回代码和测试结果 |
| T-06.2 | 子Agent失败（如Testing Agent代码语法错误），主管要求Coding Agent修复 | 最多重试2次，最后报告部分失败 |
| T-06.3 | 用户自定义一个“诗词Agent”，并调用它写一首诗 | 新Agent被调度，返回符合风格的诗词 |

## 3. 性能与压力测试
- 长上下文：持续发送10万字对话，监控内存与响应时间，不应OOM。
- 并发文件操作：100个用户同时请求整理文件夹，检查锁定机制。
- 搜索 QPS：模拟高频搜索请求，验证限流策略（默认10次/分钟/用户）。

## 4. 验收标准
- 所有 P0 用例 100% 通过，P1 用例通过率 ≥ 95%
- 性能指标符合 spec 非功能性要求
- 安全扫描无高危漏洞（文件遍历、注入等）
