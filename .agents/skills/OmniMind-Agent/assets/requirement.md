# AI Agent 需求规格说明书 (requirement.md)

## 1. 功能需求
### FR-01 多轮对话与记忆
- **FR-01.1** 系统应维护当前会话的全量消息历史，至少支持 32,000 中文字符。
- **FR-01.2** 当上下文即将超过窗口限制时，自动对较早消息生成递进式摘要，保留关键信息。
- **FR-01.3** 用户可随时要求回溯到之前对话中的任一具体信息，系统应准确引用。

### FR-02 文件系统操作
- **FR-02.1** 在用户授权的工作目录内，允许通过指令创建、删除、重命名文件/文件夹。
- **FR-02.2** 支持对指定文件夹内的文件进行批量重命名，支持模式匹配（如IMG_*.jpg → 假期_001.jpg）。
- **FR-02.3** 支持“整理下载文件夹”功能：根据文件类型/日期自动归类到子文件夹。
- **FR-02.4** 所有操作前需进行安全校验，禁止访问系统关键目录，记录操作日志。

### FR-03 联网搜索
- **FR-03.1** 当模型判断自身知识不足或用户明确要求时，自动执行联网搜索。
- **FR-03.2** 可配置至少两个搜索引擎（Bing、Google、SearchAPI），并支持故障转移。
- **FR-03.3** 搜索结果需提取摘要，并在最终回答末尾附上清晰的来源链接与序号标注。
- **FR-03.4** 提供搜索拦截规则：用户可禁止对特定域名搜索。

### FR-04 文档/链接阅读
- **FR-04.1** 支持上传文件类型：PDF、DOCX、XLSX、TXT，大小限制 50MB。
- **FR-04.2** 接受 URL 后自动抓取并解析网页正文（去除广告、导航等）。
- **FR-04.3** 支持对文档内容进行语义问答、关键点摘要，并指出答案在原文中的大致位置。
- **FR-04.4** 提供 OCR 能力处理图片型 PDF（可选启用）。

### FR-05 长时记忆
- **FR-05.1** 系统为每个用户维护私有记忆空间，存储偏好、事实、历史决策等结构化信息。
- **FR-05.2** 支持自然语言进行记忆操作：“记住我喜欢 Python 而讨厌 Java”，或“以后都用列表回复”。
- **FR-05.3** 在新会话中，Agent 应主动加载与当前话题相关的记忆，并体现在回复中（如“根据你之前喜欢的简洁风格，我列出要点”）。
- **FR-05.4** 提供记忆管理API：查询、更新、删除指定记忆条目。

### FR-06 多Agent协作
- **FR-06.1** 存在一个主管 Agent，接收用户任务并分解为子任务。
- **FR-06.2** 内置三个专业 Agent：Coding Agent、Testing Agent、Writing Agent，具备不同的系统提示和能力。
- **FR-06.3** 主管 Agent 可异步调度子 Agent，收集结果后整合回复用户。
- **FR-06.4** 用户可查看任务分解与各子Agent执行进度。
- **FR-06.5** 支持自定义 Agent 模板，用户可配置新的专业 Agent。

## 2. 用例示例
- **UC-01 整理照片**：用户说“把下载文件夹里所有图片按拍摄日期重命名并放到‘照片’文件夹”。Agent 调用文件操作模块完成。
- **UC-02 研究话题**：用户问“Transformer 的最新变种有哪些？”，Agent 判断需要联网，调用搜索 API，返回结果并引用来源。
- **UC-03 跨会话偏好**：用户昨日要求“风格简洁”，今日新对话中，Agent 自动以要点回复，并解释“记得你喜欢简洁”。

## 3. 验收标准
- 每个功能需求需通过对应的测试用例（见 test.md），覆盖正常流程、边界条件和异常场景。
